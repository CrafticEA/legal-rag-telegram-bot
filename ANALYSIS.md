# ANALYSIS.md

## 1. Цель анализа

Выявить ключевые сегменты пользователей системы, их потребности и проблемы, чтобы обосновать функциональные приоритеты продукта и подтвердить востребованность на рынке.

---

## 2. Описание продукта

Система представляет собой сервис для работы с юридическими документами в формате Telegram-бота. Пользователь может загружать документы и задавать вопросы на естественном языке.  

Основная архитектура — Retrieval-Augmented Generation (RAG), которая объединяет:

1. Семантический поиск по документам на основе эмбеддингов.
2. Генерацию ответов языковой моделью с учётом найденного контекста.

Этапы работы системы:

1. Загрузка документов пользователем.
2. Извлечение текста из документов.
3. Разбиение текста на фрагменты.
4. Построение эмбеддингов и сохранение их в векторной базе данных.
5. Поиск релевантных фрагментов по пользовательскому запросу.
6. Генерация ответа языковой моделью.

Такой подход ускоряет поиск информации и снижает риск пропуска важных деталей.

## 3. Сегментация аудитории

### 3.1 Юридические фирмы

**Характеристики:**

* Малые и средние (5–50 юристов) и крупные (>50 юристов).
* Основные задачи: анализ договоров, судебной практики, подготовка юридических заключений.

**Проблемы:**

* Риск пропуска важных деталей.
* Ограниченный доступ к дорогостоящим платформам.

**Потребности:**

* Быстрый семантический поиск по текстам.
* Локальная обработка данных без передачи на внешние серверы.

**Вывод:**
Система позволяет сократить время поиска информации на 40–70% и снизить риск ошибок при подготовке заключений.

---

### 3.2 Корпоративные юридические отделы

**Характеристики:**

* Компании с внутренними юридическими подразделениями (10–100 сотрудников).
* Контроль соблюдения законодательства, анализ рисков, подготовка внутренних регламентов.

**Проблемы:**

* Огромные архивы корпоративных документов (контракты, регламенты, внутренние политики).
* Стандартные поисковые системы ищут только по ключевым словам.
* Необходимость соблюдения конфиденциальности данных.

**Потребности:**

* Локальное хранение и обработка документов.
* Семантический поиск с учётом корпоративного контекста.
* Интеграция с внутренними системами.

**Вывод:**
Система обеспечивает безопасность данных и ускоряет поиск информации на 50–80%.

---

### 3.3 Консалтинговые и аудиторские компании

**Характеристики:**

* МСП, специалисты анализируют нормативные документы, контракты, отчёты клиентов.

**Проблемы:**

* Ручной поиск по внешним и внутренним источникам занимает слишком много времени.
* Низкая точность стандартных поисковых систем.

**Потребности:**

* Быстро находить релевантные фрагменты текста.
* Получать структурированные ответы на запросы клиентов.

**Вывод:**
Система повышает производительность консультантов и сокращает время подготовки отчётов на 30–60%.

---

## 4. Основные проблемы пользователей

1. Большие объёмы текстовой информации.
2. Низкая эффективность традиционного поиска.
3. Высокие временные затраты.
4. Риск пропуска важных деталей.

**Системное решение:** семантический поиск на основе эмбеддингов и генерация ответов языковой моделью.

---

## 5. Сценарии использования

* **Анализ договоров:** извлечение условий расторжения, обязательств сторон и штрафных санкций.
* **Поиск информации в корпоративных архивах:** быстрый поиск по десяткам и сотням документов.
* **Анализ нормативных и законодательных документов:** вопросы по содержанию с релевантными ссылками на текст.
* **Подготовка отчётов для клиентов:** структурированные ответы на сложные запросы.

---

## 6. Сравнение с существующими решениями

**Рынок:** Harvey, Hebbia, Westlaw.

**Проблемы существующих решений:**

* Высокая стоимость.
* Обработка документов на внешних серверах, что создаёт риски конфиденциальности.
* Ограниченная гибкость для внутренних документов.

**Преимущества предлагаемой системы:**

* Работа с локальными документами.
* Семантический поиск на основе эмбеддингов.
* Использование локальной языковой модели.
* Telegram-интерфейс и API интеграция.
* Быстрое масштабирование под нужды компании.

## 7. Источники

Основная аудитория: юридические фирмы, корпоративные юридические отделы и консалтинговые компании.  

Система позволяет:

* Экономить время (сокращение поиска на 40–80%).
* Снижать риск ошибок.
* Повышать эффективность работы с документами.  

Благодаря семантическому поиску и генерации ответов на основе RAG, продукт закрывает ключевые проблемы целевой аудитории и конкурентно отличается от существующих решений.
